Decision outcomes

One answer system anchored around pre-send checks.

Verdict
SAFE

Approve payout

Expected details match the provided route and the destination does not trigger a material warning.

Verdict
BLOCK

Stop before funds move

Used for clear mismatches, invalid address formats, or an obviously wrong transfer route.

Verdict
REVERIFY

Pause and confirm

Used when a destination or context needs human confirmation before approval.

Verdict
TEST FIRST

Use a low-risk trial

Used where the destination is operationally sensitive and a cautious workflow should not treat it like a normal wallet route.

Decision labels

Short labels behind the answer.

OKCore details matched and the route looks safe enough to approve.
NETWORK_MISMATCHThe selected network does not match the approved route.
ASSET_MISMATCHThe selected asset does not match the approved route.
INVALID_ADDRESSThe destination format is not valid for the selected network.
DESTINATION_REQUIRES_MEMO_OR_VENUE_CHECKThe route may depend on a venue, memo, or destination tag and needs confirmation.
DESTINATION_IS_CONTRACT_OR_APPThe destination is not a plain personal wallet and should not be treated like one.
DESTINATION_IS_BRIDGE_ROUTERThe route looks like bridge or router infrastructure, so a small test is safer.
DESTINATION_NOT_CLASSIFIEDCore details matched, but the live destination check stayed inconclusive.
DESTINATION_LOOKUP_UNAVAILABLEThe live destination check did not finish, so the result should not be treated as final approval.
UNSUPPORTED_NETWORKThe selected network is outside the current live scope.
UNSUPPORTED_ASSET_OR_NETWORKThe asset and network combination is outside the current live scope.
Why these labels matterThey keep answers consistent across dashboards, rules, support reviews, and pilot reporting.

Pilot flow

How PayeeProof fits into a real workflow.

1

Pre-send request

A payout system sends expected and provided transfer details before approval.

2

Decision + record

PayeeProof returns one clear answer together with a readable record.

3

Escalation only when needed

Human review is only needed when the answer is BLOCK, REVERIFY, TEST FIRST, or a recovery case.
