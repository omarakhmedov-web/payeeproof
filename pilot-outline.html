Why it works

Scope stays controlled, delivery stays realistic, and the control signal stays clean.

Commercial rule

Quote custom rollout work separately instead of hiding it inside the pilot.

Pilot term

A controlled 30-day risk-control pilot by default.

Default term

30 days. This keeps the commercial story clean and aligned with the monthly pilot entry.

Main objective

Validate whether PayeeProof reduces preventable transfer mistakes in one defined finance workflow before funds move.

Included

What the standard risk-control pilot covers.

  • One defined finance workflow in scope
  • Current supported network coverage
  • Pre-send risk-control checks
  • Stable verdict, risk level, and next-action language
  • Verification records
  • Basic webhook delivery where applicable
  • Email-based onboarding
  • Standard support during the pilot
  • One weekly summary
  • End-of-pilot review
Not included by default

Keep custom work separate from the standard pilot.

Custom implementation

Broad setup work, bespoke delivery, or deep integration is quoted separately when needed.

Wide rollout

Multi-team deployment, long-tail policy expansion, and business-unit complexity sit outside the standard pilot.

Custom reporting

Bespoke dashboards, heavy workflow redesign, or special SLA terms are not assumed inside the base package.

What is needed from the client

Just enough to start cleanly.

  • Business owner contact
  • Technical contact
  • Short description of the use case
  • Networks and assets in scope
  • Destination types to review
  • Current error pattern or operational risk
  • Desired action from the verdict
  • Notification destination if webhook delivery is needed
  • Target pilot start date
  • Success criterion agreed up front
End of pilot

Four clean next-step options.

  • Stop after the pilot
  • Extend for one more month
  • Move to a broader operational package
  • Quote custom implementation if rollout work is needed
