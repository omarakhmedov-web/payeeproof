Mismatch handling

Mismatch is not a dead end

The MVP should not just say “Mismatch detected.” It should explain what failed, compare expected vs provided values, and tell the user exactly what to do next.

WRONG_NETWORK Retry allowed

Provided wallet is on a different chain than requested.

Next action: resubmit using the correct network.

WRONG_ASSET Retry allowed

Requested asset does not match the provided asset.

Next action: confirm the payout asset and retry.

ADDRESS_MISMATCH Retry allowed

The submitted wallet differs from the expected destination.

Next action: review destination details before approval.

OWNERSHIP_PROOF_FAILED New request

Wallet ownership could not be verified.

Next action: start a fresh request or complete proof again.

REQUEST_EXPIRED New request

Verification session timed out.

Next action: create a new verification request.

POLICY_BLOCKED Escalate

Payout violated a configured payout rule.

Next action: review policy settings and escalate internally.
